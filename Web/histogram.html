Types of CSS

Inline CSS

Inline CSS involves adding style rules directly within an HTML element using the style attribute. This method applies styles only to the specific element.

Example:

<p style="color: blue; font-weight: bold;">This is inline CSS.</p>

Pros:

  • Quick for small changes
  • Overrides other CSS rules

Cons:

  • Hard to maintain
  • Doesn't promote reuse
  • Can clutter HTML

Internal CSS

Internal CSS is written inside a <style> tag within the <head> section of an HTML document. It affects the entire page.

Example:

<head>
  <style>
    p {
      color: green;
      font-size: 18px;
    }
  </style>
</head>

Pros:

  • Good for single pages
  • No external dependencies

Cons:

  • Not reusable across multiple pages
  • Can bloat HTML files

External CSS

External CSS uses separate .css files linked via the <link> tag in the HTML document's <head>. It allows styles to be reused across multiple pages.

Example:

<head>
  <link rel="stylesheet" href="styles.css">
</head>

Pros:

  • Reusable styles across pages
  • Keeps HTML clean and maintainable
  • Better caching and performance

Cons:

  • Requires additional HTTP requests (usually optimized)

Comparison Summary

Here is a quick summary comparing the three types of CSS:

Type Scope Maintainability Reusability
Inline CSS Single element Low None
Internal CSS Single page Medium No
External CSS Multiple pages High Yes
